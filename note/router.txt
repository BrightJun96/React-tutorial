

> vanilla js로 구현하기에는 복잡성이 있기 때문에 
react에서 라이브러리를 이용하여 구현한다.

라우팅을 하는 방법은 서버에서 경로에 맞는 템플릿 문서를 지정해 
해당 경로에 들어갔을 때 서버로부터 문서를 받아오는식으로 라우팅을 한다.

특정 경로로 이동하게 되면 서버에 요청하여 템플릿을 받아오는데 이는 화면이 깜빡거리며 잠시 
빈화면을 노출할 수 있다는 단점을 지니고 있다. 


> react-router-dom은 history api를 기반으로 하여 라우팅을 구현한다.

history API는 경로를 생성하여 추가해주고 해당 경로를 기억하게 해준다.
react-router는 이를 기반으로 해당 경로를 생성하여 경로에 맞는 컴포넌트를 할당해준다.

react에서 뿐만 아니라 라이브러리없이 이를 구현하려면 복잡성이 크지만 리액트에서는
react router library를 이용하면 훨씬 쉽게 라우팅을 구현할 수 있기 때문에 
해당 라이브러리를 사용하고 있다.


* window.history API : 경로를 생성하여 경로에 맞는 내용을 설정하고 해당 경로들을 기억하게 해준다.
* window.location API : 현재 문서의 location에 대한 정보를 담은 객체를 반환해준다.
